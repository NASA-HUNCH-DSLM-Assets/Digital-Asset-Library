# NASA HUNCH Digital Asset Workflow

This document defines the full workflow students must follow to create, document, and submit assets.

------------------------------------------------------------
1. Planning Phase
------------------------------------------------------------
1. Review NASA specifications.
2. Select required or creative asset to produce.
3. Check repository to avoid duplication.
4. Create a concept sketch if needed.

------------------------------------------------------------
2. Modeling Phase
------------------------------------------------------------
1. Model asset in Blender using real-world scale (meters).
2. Apply transforms and clean mesh.
3. Add basic materials or textures.
4. Export as FBX using default Blender settings.
5. Save the .blend file in the asset folder.

------------------------------------------------------------
3. Documentation Phase
------------------------------------------------------------
Create README.md with the following sections:

- Asset Name  
- Description  
- Real-world dimensions  
- Export formats included  
- Version  
- Author and school  
- NASA requirements met  
- Preview image reference  
- Notes for users  

------------------------------------------------------------
4. Folder Preparation
------------------------------------------------------------
Your asset folder must contain:

ASSET_NAME/  
   ASSET_NAME.fbx  
   ASSET_NAME.blend  
   preview.png  
   README.md  
   textures/ (if applicable)

------------------------------------------------------------
5. Submission via GitHub
------------------------------------------------------------
1. Fork the main repository.
2. Add your asset folder in the correct directory.
3. Commit and push your changes.
4. Open a pull request with a clear title:
   Example: "Add CTB_3_0 Required Model v1"
5. Wait for review by Artist Team or NASA Review Team.
6. Make any requested adjustments.
7. Once approved, your asset will be merged.

------------------------------------------------------------
6. Review Standards
------------------------------------------------------------
Reviewers check for:

- Correct naming conventions  
- Correct folder organization  
- Complete documentation  
- Correct scale and dimensions  
- Appropriate model complexity  
- Preview image included  
- No missing files  

------------------------------------------------------------
7. Approval and Publishing
------------------------------------------------------------
Once the pull request is approved, your asset becomes part of the official NASA Digital Asset Library and available for use by all teams.
