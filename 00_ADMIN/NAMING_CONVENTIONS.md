# NASA HUNCH Digital Asset Library
# Naming Conventions Guide

This document defines the required naming standards for all assets submitted to the NASA HUNCH Digital Asset Library.  
Consistent naming ensures that all teams can easily search, identify, and reuse models.

------------------------------------------------------------
1. General Rules
------------------------------------------------------------
1. Use only letters, numbers, and underscores.
2. No spaces.
3. No emojis or special characters.
4. Keep names descriptive but concise.
5. Use consistent prefixes for asset categories.

------------------------------------------------------------
2. Required Model Naming
------------------------------------------------------------
Format:
CATEGORY_NAME_SIZE_VERSION

Examples:
CTB_3_0_v1  
STOWAGE_SINGLE_v2  
FREEZER_M2M_v1

Rules:
- CTB sizes use the decimal format with an underscore (example: 3_0, 5_5, 10_0).
- Version numbers always start with v1.
- Required models must match NASA specifications.

------------------------------------------------------------
3. Creative Model Naming
------------------------------------------------------------
Format:
CREATIVE_NAME_VARIANT_VERSION

Examples:
Toolkit_Pouch_A_v1  
CableOrganizer_Compact_v2  
MagClamp_Lightweight_v1

Rules:
- Do not use fictional or unrealistic names.
- Creative models should be inspired by real NASA use cases.
- Include variants if multiple designs exist.

------------------------------------------------------------
4. Supporting File Naming
------------------------------------------------------------
Preview image:
preview.png

Blend file:
ASSETNAME.blend

Textures:
ASSETNAME_diffuse.png  
ASSETNAME_normal.png

Documentation:
README.md

------------------------------------------------------------
5. Folder Names
------------------------------------------------------------
Follow these formats:
CTB_3_0  
Stowage_Single  
Freezer_M2M  
Creative_ZeroG_Pouch
